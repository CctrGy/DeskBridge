DeskBridge CORE-KEYPAD UART protocol
====================================

UART:
- Serial2
- RX: PA2
- TX: PA3
- Baud: 115200
- Format: 8N1

Signal pins to CORE:
- EVENT: PA1
  Toggles every time a button event is generated. The CORE can use CHANGE interrupt.
- STATE: PB0
  Realtime OR of the 5 digital buttons. HIGH means one or more buttons are pressed.

Buttons:
- B0: PB3
- B1: PA12
- B2: PA11
- B3: PA7
- B4: PA6

Button electrical mode:
- Active HIGH
- No internal pullup

Protocol model:
- CORE is the UART master.
- KEYPAD answers text commands with text replies.
- KEYPAD does not need to stream button messages continuously.
- EVENT is raised when there is pending button data.
- CORE reads and clears pending button data with EVT? or BTN?.

Reply format:
- OK
- OK KEY=value KEY=value
- ERR REASON

Core commands:
- PING
  Reply: OK PONG

- INF?
  Reply: OK DEV=66 PROTO=TXT1 NAME=KEYPAD

- STR?
  Reply: strip-compatible state mirror.
  Current keypad-only firmware stores values sent by CORE, but does not drive a strip.

- PWR?
  Reply: OK MA=0 RAW=0 MV=0 VRAW=0 MW=0
  Current keypad-only firmware has no power sensors.

- EVT?
  Reply: OK EVT=<flags> BP=<mask> BR=<mask> BD=<mask> LB=<index> BE=<edge>
  Clears EVENT, EVT, BP, BR, LB and BE.

- BTN?
  Reply: OK BP=<mask> BR=<mask> BD=<mask> LB=<index> BE=<edge>
  Clears EVENT, BP, BR, LB and BE.

- ACT?
  Reply: OK ACT=NONE

- BAL?
  Reply: OK N=0

- BAS?
  Reply: OK B0=<id> B1=<id> B2=<id> B3=<id> B4=<id>

- BSAx,y
  Assigns action id y to button x.
  Reply: OK

- EVENT?
  Reads the enabled event mask for every button.
  Reply: OK B0=<mask> B1=<mask> B2=<mask> B3=<mask> B4=<mask>

- EVENT Bx <event> enabled
  Enables one event for one button.
  Reply: OK

- EVENT Bx <event> disabled
  Disables one event for one button.
  Reply: OK

- SCL
  Clears pending events.
  Reply: OK

- SRD
  Restores runtime defaults.
  Reply: OK

Strip-compatible mirror commands:
- SSE0 / SSE1
- SSM<mode>
- SSV<value>
- SSK<value>
- SCN<value>
- SCX<value>
- SWN<value>
- SWX<value>
- SBS<value>
- SKS<value>
- SCT<ms>
- SLT<ms>
- SRT<ms>

Button event fields:
- EVT: event flags. Button edge uses bit 9, value 512.
- BP: bitmask of buttons pressed since last clear.
- BR: bitmask of buttons released since last clear.
- BD: current button-down bitmask.
- LB: last button index, 0..4. 255 means none.
- BE: last edge. 0=none, 1=pressed, 2=released.

Button event codes:
- Bx,PRESSED
- Bx,SHORT_RELEASED
- Bx,LONG_RELEASED

Where:
- x is the button index from 0 to 4.
- These codes identify the internal event names.
- Current CORE-compatible transport reports them as masks through EVT?/BTN?.

Configurable event masks:
- 1 = PRESSED
- 2 = SHORT_RELEASED
- 4 = LONG_RELEASED
- 7 = all events enabled

Configurable event examples:
- EVENT B0 SHORT_RELEASED disabled
- EVENT B0 SHORT_RELEASED enabled
- EVENT B3 ALL disabled
- EVENT B3 PRESSED enabled

When an event is disabled:
- OneButton may still detect it internally.
- EVENT output does not go HIGH for that event.
- EVT?/BTN? will not report that event.
- STATE still reflects the physical OR of the buttons.

Examples:
- Press B0, then CORE sends EVT?
  OK EVT=512 BP=1 BR=0 BD=1 LB=0 BE=1

- Release B0 with a short click, then CORE sends EVT?
  OK EVT=512 BP=0 BR=1 BD=0 LB=0 BE=2
