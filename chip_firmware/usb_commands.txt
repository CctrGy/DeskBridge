DeskBridge USB Serial Command Shell
===================================

Version firmware: 0.2.0
Protocol serial: 2

Canal:
- Puerto Serial USB principal del firmware.
- El prompt del dispositivo es:
  db>

Formato:
- Comandos en texto plano ASCII.
- Cada comando termina con salto de linea.
- Las respuestas son texto plano legible.
- Las respuestas OK indican cambios aplicados.
- Las respuestas ERR indican comando incorrecto, argumento invalido o dispositivo no disponible.


Comandos principales
====================

ping
----
Comprueba si el firmware responde.

Respuesta:
  pong


info?
-----
Muestra informacion general del dispositivo.

Incluye:
- nombre del dispositivo
- version firmware
- version del protocolo serial
- estado USB
- estado CDC
- uptime


usb?
----
Muestra informacion USB.

Incluye:
- VID
- PID
- bcdDevice
- mounted
- VBUS
- estado CDC


program?
--------
Muestra informacion del programa cargado en el microcontrolador.

Incluye:
- nombre
- version firmware
- semver
- protocolo serial
- build timestamp


sys?
----
Muestra informacion local del chip.

Equivale a:
  sys local

Incluye:
- chip
- frecuencia CPU
- temperatura interna
- VDDA
- razon de reset
- uptime


sys local
---------
Muestra informacion local del chip.


sys periferics
--------------
Muestra el estado de perifericos del I2C de sistema.

Tambien acepta:
  sys peripherals

Actualmente lista:
- DS3231
- AT24C32


bus?
----
Muestra informacion general del bus de sensores.

Incluye:
- time_read
- samples
- estado ENS160
- estado AHT21
- estado VEML7700
- estado SCD41
- estado SHT21


bus scan
--------
Escanea el bus I2C de sensores.

Devuelve:
- cantidad de direcciones encontradas
- lista de direcciones I2C


bus time_read
-------------
Muestra el intervalo actual de lectura de sensores.


bus time_read [value]
---------------------
Configura el intervalo de lectura de sensores.

Formatos aceptados:
- 1000
- 1000ms
- 1.5s

El firmware limita el valor al rango permitido.


bus samples
-----------
Muestra el numero actual de muestras usadas para promedio.


bus samples [value]
-------------------
Configura el numero de muestras usadas para promedio.

El firmware limita el valor al rango permitido.


bus sensor
----------
Lista los sensores disponibles.

Sensores:
  0 : ENS160
  1 : AHT21
  2 : VEML7700
  3 : SCD41
  4 : SHT21


bus sensor [sel]
----------------
Muestra estado y ultima medicion de un sensor.

Seleccion por indice:
  bus sensor 0
  bus sensor 1
  bus sensor 2
  bus sensor 3
  bus sensor 4

Seleccion por nombre:
  bus sensor ENS160
  bus sensor AHT21
  bus sensor VEML7700
  bus sensor SCD41
  bus sensor SHT21


rtc?
----
Muestra el estado del RTC DS3231.

Incluye:
- estado RTC
- lost_power


rtc get
-------
Muestra fecha y hora actual del RTC.

Formato:
  YYYY-MM-DDTHH:MM:SS


rtc set [value]
---------------
Ajusta fecha y hora del RTC.

Formatos aceptados:
  YYYY-MM-DDTHH:MM:SS
  YYYY-MM-DD_HH:MM:SS

Ejemplos:
  rtc set 2026-05-30T18:45:00
  rtc set 2026-05-30_18:45:00


Comandos eliminados
===================

sensor?
sensor [sel]

Estos comandos ya no existen como comandos raiz.
Usar en su lugar:
  bus sensor
  bus sensor [sel]
